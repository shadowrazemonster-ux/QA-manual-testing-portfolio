# Test Plan - Example.com

## Project Overview
This test plan covers testing of the Example.com website to ensure functional and UI correctness.

## Scope
- Login functionality
- Registration
- Profile update
- Password reset
- UI elements and navigation

## Test Approach
- Manual testing using predefined test cases
- Bug reporting in the bug-reports folder
- Checklist verification for critical flows
- Regression testing after bug fixes

## Test Deliverables
- Test cases stored in the test-cases/ folder
- Bug reports stored in the bug-reports/ folder
- Checklists (if applicable)

## Roles and Responsibilities
- QA Engineer: Execute test cases, document bugs, maintain test plan
- Developer: Fix reported bugs, provide clarification if needed
- Project Manager: Review test plan and approve test cycles

## Schedule
- Test case creation: Completed
- Bug reporting: Ongoing
- Review and finalization: Within 1 week

## Entry and Exit Criteria
- Entry Criteria: Test environment ready, all test cases documented
- Exit Criteria: All planned test cases executed, critical bugs fixed, sign-off obtained

## Tools
- Browser: Chrome
- GitHub: Repository and Issue tracking
- Markdown: Documentation of test cases and bugs
- Optional: Screenshots or recording tools for evidence

## Notes
- This test plan is designed for Junior QA portfolio purposes
- Focus is on demonstrating structured testing and reporting
